# fuck-wjx (问卷星速填)

[English](./README.md) | 简体中文

参考了   https://github.com/Zemelee/wjx   的仓库,请不要忘记给这位佬给出一个star⭐

本项目纯粹就是瞎搞了一些新功能，提供了图形界面用于配置题目分布、权重、填空答案等，用于自动填写问卷星问卷并批量提交（苦逼大学生们有救了哈哈）

**注意：此项目仅供学习与测试目的使用，请确保对目标问卷有自动提交的授权，避免滥用或违反他人服务条款，严禁污染他人的问卷数据**

---

**主要特性**

- 💻具有友好的GUI界面，零门槛，无需编程基础，认得汉字就能用
- ⭐具备本地二维码解码功能：可直接上传问卷二维码图片自动识别出问卷链接
- 😄免费无限量，不再求助群聊朋友圈
- 🧑‍🤝‍🧑可以自动解析题目内容，并通过配置向导自己预设答案，调整选项概率和比例
- 🚀减少提交后的等待时间，提升刷问卷数量的速度
- 🎭全真模拟模式：模拟真人打字速度和鼠标移动，让提交更加自然逼真
- 🌍随机IP提交：绕过IP限制、智能验证
- ⏸️随时停止：可随时优雅地停止执行而不丢失进度
- 📁支持保存问卷，方便多次复用同一份问卷的答案策略

**关键配置说明**

在对下列选项进行微调前，务必先通过配置向导解析当前问卷。每次修改问卷链接或策略后，界面会自动刷新题目预览。

- `问卷链接`
  - 粘贴完整的问卷星链接（示例：`https://v.wjx.cn/vm/...`）。
  - 也可以点击「上传二维码」，由程序自动解码并填入链接。
  - 链接变更后，请确认左侧题目列表同步更新，以免解析到了错误的问卷。

- `目标份数`
  - 需要自动填写并提交的总份数。
  - 首次运行建议设置 3～5 份，确认逻辑无误后再逐步加大。
  - 界面会显示剩余份数，过程中可暂停或关闭窗口以提前停止。

- `浏览器数量`
  - 控制同时打开的 Playwright 浏览器实例数量（并发数量）。
  - 数值越大吞吐越高，但同时占用更多 CPU/RAM；大多数台式机建议保持在 2～5 之间。
  - 长时间运行时可结合任务管理器监视资源使用，必要时调小该值。

- `分布方式`
  - `完全随机`：从所有候选项随机选择答案，适合快速测试。
  - `自定义权重`：为每个选项分配权重，程序会按照比例概率抽取结果。
  - 在 GUI 中可拖动滑块或输入百分比，界面会实时展示当前权重分布。

- `题目列表区`
  - 左侧列出解析到的题目及其类型（单选、多选、矩阵、填空等）。
  - 点击题目后，右侧面板会显示其具体配置：答案预设、权重、固定选项等。
  - 多选题 / 矩阵题可针对每一行列分别设置权重或锁定状态，确保复杂布局得到合理控制。

- `全选` / 批量操作
  - 表头左上角的复选框是「全选」按钮，可一次性选中或取消所有题目。
  - 勾选题目后，使用下方或右键菜单的按钮：
    - 「批量删除」：清除所选题目的自定义配置但不影响原始问卷。
    - 「批量编辑」：对多个题目同时应用相同的分布或答案策略。

- `填空题 / 主观题配置`
  - 填空题可以设置一个固定答案，或提供一个候选答案列表，程序会随机选择一个填写。
  - 主观题（长文本）可配置一段固定文案，也可预设多个回复，由程序随机挑选，避免每份答卷都雷同。
  - 可在预览中确认换行与格式，确保提交内容符合预期。

- `自动配置问卷`
  - 每次更换问卷链接后点击此按钮，程序会访问页面、识别题型并提供默认配置。
  - 某些问卷可能要求登录、填写验证码或设置访问权限，如解析失败请先排查这些条件。

- `全真模拟设置`
  - 启用「全真模拟模式」可模拟真人填写行为，包括自然的打字速度和鼠标移动轨迹。
  - 设置总时长（时:分:秒）来分散提交时间，避免短时间内大量提交被检测。
  - 此模式大幅降低被识别风险，但完成速度较慢。

- `随机IP提交`
  - 勾选「启用随机IP提交」可绕过问卷的IP限制。
  - 适用于有严格IP限制的问卷，但会产生额外成本。

- `失败过多自动停止`
  - 启用后，当连续失败次数过多时会自动停止执行。
  - 防止问卷失效或结构变化时浪费资源。

- `开始执行`
  - 确认所有参数无误后，点击「开始执行」即可按照设定自动填写并提交问卷。
  - 右侧日志会显示进度、成功数量与错误记录，有助于调试。
  - 使用「停止」按钮可随时优雅地终止执行——系统会等待活跃线程安全完成后再退出。
